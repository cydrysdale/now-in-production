# Claims Ledger

Use this to document every claim the campaign is allowed to make, with approved wording, proof sources, and freshness windows.

Do not use claims in campaign copy unless they appear here with `approved` status.

## Filling Out This Template

If this template has blank fields, help the user complete it before doing any other work. Use the interview guide below.

If this template is already complete, skip the interview and use it as campaign context.

### Interview Guide

Ask questions in small groups. Wait for answers before continuing. Use plain language.

**Start with:** "I'll help you build the claims ledger for this campaign. I'll ask about the claims you need to make, and for each one I'll help you document the approved wording, evidence, and any restrictions. Let's start."

**Group 1 — Scope:**

- What campaign is this claims ledger for?
- What are the main claims or statements this campaign needs to make? List them in plain language — we will formalize each one.
- Are any of these claims about product performance, market position, or competitive comparisons?

**Group 2 — For each claim the user listed, ask:**

- What is the approved way to say this? Is there exact wording that must be used?
- Is there any way of saying this that is not allowed?
- Does this claim need a qualifier (e.g., "for enterprise clients" or "as of Q1 2025")?
- What is the evidence for this claim? Where does it come from (document, report, product page)?
- Who approved this claim and when?
- Does this claim have an expiration date or need to be refreshed?
- Which channels is this claim allowed in? All of them, or only specific ones?

**Group 3 — Proof anchors:**

- Beyond specific claims, are there proof points this campaign should use (metrics, coverage stats, client outcomes, analyst mentions)?
- For each proof anchor: what is the approved summary, where is the source, and when does it need to be refreshed?

**Group 4 — Risks and gaps:**

- Are there any claims the team wants to make but cannot yet? What is blocking them?
- Are there any claims that sound true but are not approved to use?
- Are there any areas where the evidence is thin or the wording is still under review?

After the interview, output the completed ledger using the template format below. Mark any claims where the user was uncertain about approval status as `conditional` and add them to Open Questions.
