# Copy AI Packet RACI — Draft

> **Status: Draft exploration — not an approved process.**
> This RACI is part of an active workflow exploration. It proposes responsibility assignments for discussion.

Roles:

- `CM` = Campaign manager
- `CW` = Copywriter
- `CE` = Copyeditor
- `PROD` = Production
- `LEADS` = B&C or Marketing leads

| Activity | CM | CW | CE | PROD | LEADS |
| --- | --- | --- | --- | --- | --- |
| Intake and request framing | A/R | C | C | R | I |
| Define audience, objective, CTA, and campaign direction | A/R | C | C | C | I |
| Translate campaign direction into messaging strategy | C | A/R | C | C | I |
| Define editorial quality bar and anti-patterns | I | C | A/R | C | I |
| Approve proof points and claims direction | C | A/R | C | C | I |
| Approve examples for reuse | I | A/R | C | C | I |
| Build packet structure and workflow | I | C | C | A/R | I |
| Maintain metadata, versioning, and file structure | I | I | I | A/R | I |
| Run readiness and QA gates | I | C | C | A/R | I |
| Approve campaign spine | A/R | C | C | C | I |
| Review generated drafts for editorial quality | I | C | A/R | I | I |
| Log repeated rewrite patterns | I | I | A/R | C | I |
| Convert repeated issues into packet updates | I | C | C | A/R | I |
| Approve packet as a shared standard | I | C | C | C | A/R |
| Publish and maintain packet library | I | I | I | A/R | I |

Notes:

- `A` = Accountable
- `R` = Responsible
- `C` = Consulted
- `I` = Informed

For one-off campaign use, leads are typically informed rather than approving.

For shared-standard rollout, lead approval is required before publication as the default approach.
